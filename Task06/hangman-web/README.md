# Task06 - Игра "Виселица" (браузерное приложение на JavaScript)

## Описание проекта

Лабораторная работа №6: реализация игры «Виселица» в виде браузерного приложения на языке JavaScript (ES6).  
Это браузерная версия игры из предыдущих лабораторных работ, но **без работы с базой данных**:

- игра запускается прямо в веб-браузере;
- интерфейс реализован с помощью HTML и CSS;
- вся логика игры написана на чистом JavaScript (ES-модули);
- данные о партиях не сохраняются (в отличие от версий на PHP/SQLite и ORM RedBeanPHP).

В следующей лабораторной работе (Task07) на эту же основу будет добавлена работа с локальной NoSQL-базой IndexedDB.

### Правила игры

Компьютер загадывает слово из 6 букв английского алфавита.  
Игрок по одной вводит буквы с клавиатуры:

- если буква есть в слове — показываются все её вхождения;
- если буквы нет — увеличивается число ошибок и дорисовывается часть человечка на виселице;
- отображаются текущее состояние слова, количество ошибок и уже использованные буквы.

Игрок выигрывает, если угадывает все буквы до того, как будут допущены 6 ошибок.  
При 6 ошибках игра заканчивается поражением.

### Модули приложения

Код организован по тому же принципу, что и в консольной версии:

- `src/game.js` — игровая логика (модель):
  - хранит список возможных слов (массив внутри модуля);
  - выбирает случайное слово;
  - хранит состояние (ошибки, угаданные буквы, статус игры);
  - проверяет угадывание и переход в состояния win/lose.
- `src/view.js` — слой представления:
  - работа с DOM-элементами;
  - отображение виселицы, слова, ошибок и сообщений;
  - чтение ввода пользователя.
- `src/controller.js` — контроллер и точка входа:
  - связывает `HangmanGame` и `View`;
  - инициализируется после загрузки `DOMContentLoaded`;
  - обрабатывает события (старт игры, ввод буквы, новая игра).
